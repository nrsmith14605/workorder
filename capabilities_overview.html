Platform Overview
A Smarter Way to Manage District Work Orders

The WCSC Work Order Management System is a purpose-built web application that centralizes and streamlines how Warrick County School Corporation staff submit, track, and resolve facilities and technology service requests. Built entirely around the district's existing Google Workspace accounts, the system requires no separate logins, no new passwords, and no installation — staff access it from any browser on any device.

From a teacher submitting a broken projector request to a maintenance manager dispatching a crew to a plumbing issue, every interaction flows through a structured, role-based pipeline with automatic email notifications, photo documentation, and a full audit trail.

Access Control
Role-Based Permissions

Every user account is assigned one of nine roles that precisely control what they can see and do within the system. Roles are managed by an administrator and take effect at next login — no configuration required by the user.

Role Code Title Capabilities & Scope
A Administrator Full system access. Manages all users, views and administers all orders of both types, accesses all reports, configures role assignments and building associations.
MM Maintenance Manager Approves and rejects maintenance orders, assigns maintenance workers, marks orders complete, generates maintenance-scoped PDF reports. Sees all maintenance orders district-wide.
MT Technology Manager Approves and rejects technology orders, assigns building technicians, marks orders complete, generates technology-scoped PDF reports. Sees all technology orders district-wide.
BP Building Principal Views all orders submitted from their assigned building. Approves or rejects orders from their building before they route to the appropriate manager.
BT Building Technician Receives technology order assignments. Views and works orders assigned to them across their associated buildings. Mobile-optimized experience available.
BC Building Custodian Receives maintenance order assignments. Views only their personally assigned orders. Can add notes, attach photos, and mark orders complete from the field.
BM Building Maintenance Receives maintenance order assignments. Equivalent capabilities to Building Custodian — notes, photos, and completion from the field via mobile interface.
MW Maintenance Worker District-wide maintenance worker. Views only assigned orders. Adds notes and photos from the field, marks work complete via mobile. Not building-scoped.
U General User Teachers, office staff, and all other district employees. Submits maintenance and technology work orders. Views only their own submitted orders and their current status.
Building-Scoped Roles Building Principals (BP) and Building Technicians (BT) are associated with one or more specific buildings at the time of account creation. Their view of the system is automatically filtered to those buildings — they never see work orders from other sites.
Account Activation & Deactivation Administrators can activate or deactivate any user account at any time without deleting it. Deactivated users lose access immediately upon their next login attempt while their historical data and assignments remain intact in the system.

Platform Capabilities
Core Features
Google Single Sign-On
Staff sign in with their existing district Google Workspace accounts — no separate usernames or passwords. Access is locked to the @warrick.k12.in.us domain, preventing unauthorized access by default.
Intelligent Order Submission
Guided submission form with required fields for order type, building, room number, problem category, description, priority level, and optional time-window scheduling. Users cannot submit incomplete orders.
Multi-Photo Uploads
Work orders support multiple photo attachments at submission time. Field workers can also add photos while updating in-progress orders. All images are stored securely and viewable in the order detail page.
Automated Email Notifications
The system automatically sends email alerts at each workflow stage: submission confirmation to the requester, approval/rejection notices, assignment notifications to workers, and completion confirmation — all without manual action.
Worker Assignment System
Managers can assign one or more workers to a single order from a filtered list of eligible staff. Multiple assignments are supported when a job requires a crew. Assigned workers receive immediate email notification and the order appears on their dashboard.
Live Notes & Activity Log
Any authorized user can append timestamped notes to an order at any point in its lifecycle. Notes are displayed in chronological order on the detail page, creating a running activity log for every work order.
Print-Ready Work Orders
Every work order has a dedicated print view that formats all order details, notes, photos, and assignment history into a clean, professional document ready for physical filing or distribution without extra formatting.
Building & Room Routing
Orders are tagged with building code and room number at submission. The system uses this to automatically route visibility — principals see their building, technicians see their sites, and district-wide staff see everything.
Real-Time Notification Bell
An in-app notification bell alerts users to new activity on their orders without requiring them to refresh the page — status changes, new assignments, and approvals appear as unread badge counts in the navigation bar.
Searchable Order Numbers
Every submitted order receives a unique, formatted work order number (e.g., WO-000042) that can be used to directly navigate to any order via URL. Numbers are sequential, permanent, and consistent across printed and digital records.

Analytics & Documentation
PDF Reporting Suite

Administrators, Maintenance Managers, and Technology Managers have access to a built-in reporting engine that generates branded, print-quality PDF documents on demand. All reports are scoped to the generating user's role — a Maintenance Manager sees only maintenance data, a Technology Manager only technology data, and an Administrator sees everything.

Active Maintenance Orders
A complete list of all open maintenance work orders, sorted by priority then age. Includes building, room, problem type, assigned workers, status, and days open.
Active Technology Orders
Equivalent report for all open technology service requests. Displays current handler, assignment status, and time open for each unresolved ticket.
Aging Report
Identifies orders that have been open beyond a configurable threshold (default: 14 days). Color-coded age indicators highlight items at risk of falling through the cracks.
Workload Distribution
Shows how completed orders were distributed across all workers over a configurable time period (30, 60, or 90 days / 1 year). Supports accountability and workload balancing.
Completed Orders by Staff
Generates a history of all orders completed by a selected employee over a chosen period. Useful for performance reviews, documentation, and tracking individual output.
Current Staff Workload
Multi-employee snapshot of all currently open, assigned orders. Supports side-by-side comparison of active workloads across an entire team to help managers allocate resources.
Report Format & Branding
Every generated report is a fully formatted PDF document branded with the Warrick County School Corporation logo, organization name, and contact information in the header. Reports include a filter summary line, generation timestamp, the generating user's name, and a footer on every page. All documents are generated instantly and download directly to the user's device — no external printing service or third-party software required.

Field & Remote Access
Mobile Experience

Field staff — custodians, maintenance workers, and building technicians — work in the field, not at a desk. The system includes a dedicated mobile interface that automatically activates when these roles access the system from a phone or tablet.

Progressive Web App (PWA)
  • Install to home screen on iOS and Android devices — works like a native app, no App Store required
  • Standalone display with no browser chrome — full-screen experience once installed
  • Automatic redirect — field-role users on mobile devices go directly to the mobile dashboard without any extra steps
  • Desktop escape — a "Switch to Desktop" option is always available for field workers who need full access
  • District branding — WCSC gear icon appears on the user's home screen after installation
Mobile Dashboard Features
  • Priority-sorted card view — Urgent orders surface first, then High, Mid, Low; within priority, oldest orders appear first
  • Status filtering — toggle between In Progress, Approved, and Completed views with a single tap
  • In-field notes — add text notes to an order from the job site in real time
  • Photo upload from device — attach photos directly from the camera or photo library
  • Mark complete — field workers can mark their assigned orders complete from the mobile view; managers confirm from the desktop
  • Role-aware scope — workers see only their assigned orders; Maintenance Manager sees all open maintenance orders
No App Store. No IT Deployment. No Updates to Manage.
Because the mobile experience is a Progressive Web App delivered through the browser, there is nothing to install through the App Store or Google Play. When the system is updated, every user gets the new version automatically the next time they open it — no forced update prompts, no device management configuration, and no MDM policy changes required.
Access & Data Protection
Security Architecture
Google Workspace Authentication
All login is handled by Google's OAuth 2.0 infrastructure. The system never stores district passwords. If a user's Google account is suspended by the district, they immediately lose system access without any additional action required from the administrator.
Domain-Locked Access
The system is hard-coded to accept only @warrick.k12.in.us email addresses. Attempts to authenticate with personal Gmail accounts or addresses from any other domain are rejected at the login stage, before any session is created.
Role-Based Data Isolation
Every database query is filtered by the authenticated user's role and building assignment. A general user cannot access the URL of another user's work order. A principal cannot see another building's orders. Access is enforced server-side on every page load, not just on navigation.
Session Management
Sessions are validated on every page request. Unauthenticated requests are immediately redirected to the login screen. Role mismatches on restricted pages (e.g., a general user attempting to access the admin panel) redirect rather than expose an error or partial content.
